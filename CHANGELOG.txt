Altimeter Desktop App - BLE Export Changes
=========================================

2026-01-04

- Added support for an optional `sessionId` field in the device's FILEINFO
  response and exposed it in the desktop logs for easier debugging of
  individual export attempts.
- Tightened validation for chunked BLE exports (FINFO/FGET): after the
  full CSV dump is assembled, the desktop now checks that the number of
  parsed data rows exactly matches `totalSamples` reported in FILEINFO.
  If there is any mismatch (for example due to a mid-transfer BLE
  disconnect or truncated chunk), the export is treated as a failure:
  an "Export Incomplete" warning is shown and no partial dataset is
  loaded into the plots or tables.
- Left the legacy single-shot dump path unchanged but re-used the same
  CSV cleaning pipeline so behavior is consistent across both paths.
